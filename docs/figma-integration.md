# Интеграция дизайна Figma в CRM (Next.js + Tailwind v4)

Цель: предсказуемо переносить макеты из Figma в этот репозиторий без «рассыпающегося» UI и без копирования сырого кода из Figma Make.

## Почему «просто вставить макет» не работает

| Источник | Что это даёт | Ограничение |
|----------|----------------|------------|
| **Figma Design** (фреймы, компоненты, variables) | Цвета, отступы, типографика, структура | Нужно руками/агентом привязать к React и вашим классам (`crm-*`, `glass-*`). |
| **Figma Make** | Прототип на React (часто shadcn + motion) | Другой стек, другие зависимости; годится как **референс**, не как drop-in. |
| **MCP Figma в Cursor** | `get_design_context`, скриншот, иногда исходники Make | Нужно **адаптировать** под [`app/globals.css`](../app/globals.css) и [`components/`](../components/). |

Итог: интеграция = **контракт токенов + повторно используемые классы + один раз перенесённые паттерны**, а не автоматическая замена страниц.

---

## Уровень 1 — Токены (обязательный минимум)

1. В Figma заведите **Variables** (или Styles) для: фона, текста, акцента, радиуса, тени, основных отступов.
2. В Dev Mode откройте переменные и зафиксируйте значения в таблице соответствия с проектом.

Перенос в код:

- Основной файл токенов приложения: [`app/globals.css`](../app/globals.css) (`:root`, классы `crm-*`, `make-glass-*`).
- При обновлении макета **сначала** обновляете переменные в Figma, затем **одно место** в `globals.css`, а не десятки страниц.

Рекомендуемая таблица (пример):

| Figma variable | CSS в проекте |
|----------------|----------------|
| `accent/default` | `--accent`, `--accent-strong` |
| `surface/page` | фон `body` / `.crm-make-shell` |
| `surface/card` | `.make-glass-card-static` / `.crm-surface` |

## Уровень 2 — Code Connect (лучшая связка «компонент в Figma = компонент в коде»)

[Code Connect](https://www.figma.com/developers/code-connect) связывает **экземпляры компонентов в файле Figma** с **реальными компонентами** в репозитории (например `components/layout/Sidebar.tsx`).

**Зачем:** при вызове MCP `get_design_context` агент чаще предложит **ваши** компоненты и классы, а не абстрактный Tailwind с нуля.

**Шаги (кратко):**

1. В Figma включите Code Connect для команды/файла (зависит от тарифа).
2. Установите CLI (`npx figma connect`) и создайте рядом с компонентом файл вида `ComponentName.figma.tsx` по документации Figma.
3. Запустите публикацию маппинга — после этого Dev Mode может отдавать привязку к коду.

Детали меняются у Figma; опирайтесь на актуальную статью в Help Center и репозиторий `figma/code-connect`.

## Уровень 3 — Рабочий процесс с Cursor + MCP (у вас уже есть)

В Cursor подключён официальный **Figma MCP**. Типичный цикл для новой страницы:

1. Дать агенту **ссылку на фрейм** (`figma.com/design/...?node-id=...`) или **Figma Make** (`figma.com/make/...`).
2. Агент вызывает **`get_design_context`** (и при необходимости скриншот).
3. Вы явно просите: **не добавлять новые npm-зависимости**, использовать существующие [`crm-ui` паттерны](../.cursor/rules/crm-ui-design-workflow.mdc) и [`globals.css`](../app/globals.css).
4. Результат — правки в `components/` и `app/(crm)/`, а не новый параллельный «демо-слой».

Для **Make** MCP часто возвращает список исходников (`App.tsx`, стили); это ориентир для структуры, не для прямого копирования в прод.

## Что закрепить в команде (чеклист перед мержем)

- [ ] Цвета/радиусы не «разъехались» — сверка с `:root` в `globals.css`.
- [ ] Нет лишних библиотек ради внешнего вида (политика проекта — Tailwind + свои классы).
- [ ] Ключевые оболочки: [`AppShell`](../components/layout/AppShell.tsx), [`Sidebar`](../components/layout/Sidebar.tsx), [`Header`](../components/layout/Header.tsx) — единый источник для навигации и фона.
- [ ] Страницы без макета в Figma **не ломаем** — только общий shell/токены при необходимости.

---

## Если редизайн «не виден» локально

1. Один процесс dev на порту (не два `next dev`).
2. После смены ветки / больших правок CSS: удалить `.next`, перезапустить `npm run dev` или `npm run dev:vercel`.
3. Убедиться, что смотрите **ту же копию репозитория**, где есть коммит с правками (Vercel показывает только запушенный код).
